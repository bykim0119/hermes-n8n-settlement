# 정산 대사 자동화 — hermes(MCP) × n8n

## 배경

이커머스 운영에서는 정산 주기마다 거래처가 전달한 정산 내역과 실제 판매 데이터를 비교하는 **정산 대사** 작업이 필요합니다. 거래처가 정산한 수량과 내부 판매 수량이 일치하는지 확인하고, 차이가 발생한 경우 취소·반품·덤·사은품 등의 원인을 다시 추적해야 합니다.

거래처가 많아질수록 이 작업에 드는 시간도 빠르게 늘어납니다. 대부분의 정산 건은 문제가 없지만, 운영자는 정상적인 건까지 하나씩 확인해야 합니다. 그 과정에서 실제로 확인이 필요한 소수의 예외가 반복 작업 속에 묻히기도 합니다.

규칙은 비교적 단순하지만 처리해야 할 데이터는 많고, 예외는 드물지만 반드시 확인해야 합니다. 이 프로젝트는 이러한 반복적인 비교 작업을 자동화하고, 운영자가 판단해야 할 불일치 건에 집중할 수 있도록 만드는 것에서 시작했습니다.


## 접근

이 프로젝트는 정산 대사 작업을 성격에 따라 두 단계로 나눕니다.

* **숫자를 비교해 불일치를 찾는 반복 작업은 규칙으로 처리합니다.** 거래처의 정산 내역과 내부 판매 데이터를 거래처별로 비교한 뒤, 수량이 일치하지 않는 건만 추려냅니다. 정해진 기준에 따라 계산하면 되는 작업이므로 AI를 사용하지 않습니다.
* **수량이 왜 일치하지 않는지 확인해야 하는 건만 AI 에이전트가 조사합니다.** Hermes 에이전트는 SQLite 데이터베이스를 SQL로 직접 조회해 취소·반품·덤·사은품 수량을 확인합니다. 차이가 모두 설명되면 **정상**, 설명되지 않은 수량이 남으면 **사람의 확인이 필요한 건**으로 분류합니다.

즉, 모든 과정에 AI를 사용하는 것이 아니라 판단이 필요한 구간에만 제한적으로 적용하고, 단순하고 명확한 계산은 규칙에 맡깁니다. 이러한 역할 분담을 n8n 워크플로우 위에서 한눈에 확인할 수 있도록 구성한 것이 이 프로젝트의 핵심입니다.


## 구조

```
hermes-n8n-settlement/
├─ docs/
│  ├─ specs/                   설계 문서
│  ├─ plans/                   구현 계획
│  └─ investigate-internals.md 에이전트 조사 노드 내부 동작(구조도)
├─ data/        CSV·SQLite (주문상세·정산마감), 정답지(비공개)
├─ scripts/     load_to_sqlite · reconcile · investigate · report · serve · run_all
├─ workflows/   n8n 워크플로우 export (json)
└─ reports/     파일 리포트 출력 (git 미추적)
```

## 파이프라인

CSV → SQLite 적재 → **규칙 대조**(불일치만 추출) → 불일치 건별 **hermes 조사**(SQL로 취소·반품·덤 확인)
→ 판정(정상 / 에스컬레이션) → 취합 → 보고(마크다운 파일)

## 두 가지 n8n 워크플로우


| 파일 | 방식 | n8n 캔버스에 보이는 것 |
|---|---|---|
| `settlement_skeleton.json` | **한 번에 처리** — 서버(`/run`)가 대조·조사·저장까지 전부 | 시작 → 서비스 호출 (노드 2개) |
| `settlement_full.json` | **단계별 처리** — n8n이 직접 오케스트레이션 | 시작 → 규칙 대조 → (불일치 3건 분기) → 에이전트 조사 → 취합 (노드 4개) |


## 실행 방법

```bash
# 1) HTTP 서비스 기동 (기동 시 CSV→SQLite 적재 포함)
python3 scripts/serve.py            # http://127.0.0.1:8787

# 2) 아래 셋 중 하나로 실행
#  2-a) n8n에서 workflows/settlement_skeleton.json import → 실행 (결과: reports/*.md)
#  2-b) n8n에서 workflows/settlement_full.json    import → 실행 (결과: n8n 화면의 취합 노드)
#  2-c) CLI 단독 실행 (디스코드 게시하려면 웹훅 환경변수 지정)
DISCORD_WEBHOOK_URL=... python3 scripts/run_all.py

# 3) 결과 확인: reports/ 의 최신 .md (+ 웹훅 지정 시 디스코드 채널)
```
